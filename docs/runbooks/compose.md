# Compose Runbook

Step-by-step instructions for launching and verifying the full PlantPal
EX3 stack locally.

---

## 1. First-time setup

```bash
# Clone and enter the repo (if not already)
cd /path/to/PlantPal

# Copy the environment template
cp .env.example .env
```

The defaults in `.env.example` work for local use. The only optional
change is adding `GOOGLE_API_KEY` + `GOOGLE_GEMINI_MODEL` if you want
the AI service to use Google Gemini instead of the rule-based fallback
(see section 5).

---

## 2. Start the stack

```bash
sudo docker compose -f compose.yaml up --build
```

Add `-d` to detach and run in the background:

```bash
sudo docker compose -f compose.yaml up --build -d
```

### What starts

| Container | Role | Port |
|-----------|------|------|
| `plantpal-redis` | Cache, rate-limit counters, idempotency | 6379 |
| `plantpal-backend` | FastAPI + SQLite | 8000 |
| `plantpal-ai` | AI care advisor | 8001 |
| `plantpal-frontend` | Streamlit dashboard | 8501 |
| `plantpal-worker` | Background health-refresh loop | — |

### Verify all services are healthy

```bash
sudo docker compose -f compose.yaml ps
```

All five should show `healthy` or `running`. Also confirm individually:

```bash
curl http://localhost:8000/health   # {"status":"ok","service":"plantpal-backend"}
curl http://localhost:8001/health   # {"status":"ok","service":"plantpal-ai-advisor"}
```

---

## 3. Seed sample data

```bash
sudo docker compose -f compose.yaml exec backend uv run python seed.py
```

Expected output:

```
  + Monstera (id=1)
  + Snake Plant (id=2)
  ...
Seeded 8 plants.
Seeded 30 care events.
```

Idempotent — safe to run again, skips if plants already exist.

Open the dashboard at http://localhost:8501 — you should see 8 plant
cards with health pills and watering rings.

---

## 4. Verify Redis cache and rate-limit headers

After seeding, hit the plant list endpoint:

```bash
curl -i http://localhost:8000/plants/ 2>&1 | grep -i "x-ratelimit\|http/"
```

Expected:

```
HTTP/1.1 200 OK
x-ratelimit-limit: 60
x-ratelimit-remaining: 59
```

Hit it again — the second response is faster (cache hit). Confirm
the Redis keys:

```bash
# Rate-limit counters
sudo docker compose -f compose.yaml exec redis redis-cli keys 'rate:*'

# Cached plant list
sudo docker compose -f compose.yaml exec redis redis-cli keys 'plants:*'
```

## 5. AI advice

### Rule-based (no setup — always works)

```bash
curl http://localhost:8000/plants/1/advice | python3 -m json.tool
```

Response includes `"source": "rule-based"`.

### Google Gemini (optional — real LLM)

**Get a free API key at** https://aistudio.google.com/ (click
**Get API key** → **Create API key**).

Add to `.env`:

```ini
GOOGLE_API_KEY=your-key-here
GOOGLE_GEMINI_MODEL=gemini-1.5-flash
```

Rebuild only the AI service (fast — doesn't touch other containers):

```bash
sudo docker compose -f compose.yaml up --build ai_service
```

Then fetch advice again:

```bash
curl http://localhost:8000/plants/3/advice | python3 -m json.tool
```

The response now shows `"source": "gemini"` and the tips are
generated by the LLM based on the plant's actual data.

**In the dashboard**, click the **🤖** button on any card to see
the advice panel inline.

## 6. Authentication

### Get an editor JWT

```bash
TOKEN=$(curl -s -X POST http://localhost:8000/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=gardener&password=plantpal" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

echo "Token: ${TOKEN:0:30}..."
```

### Create a plant (requires editor token)

```bash
curl -s -X POST http://localhost:8000/plants/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Test Plant","species":"Demo sp.","location":"Office"}' \
  | python3 -m json.tool
```

### Confirm 401 without token

```bash
curl -s -X POST http://localhost:8000/plants/ \
  -H "Content-Type: application/json" \
  -d '{"name":"Fail","species":"x"}' | python3 -m json.tool
# {"detail":"Not authenticated"}
```

> The Streamlit dashboard handles auth automatically — it logs in with
> `DEFAULT_EDITOR_USERNAME` / `DEFAULT_EDITOR_PASSWORD` on startup and
> stores the token in session state.

## 7. Async refresher

The worker container runs this loop automatically every 5 minutes.
To trigger manually:

```bash
# From outside Docker (run from repo root, backend venv must exist)
backend/.venv/bin/python scripts/refresh.py --limit 20

# From inside Docker
sudo docker compose -f compose.yaml exec worker \
  uv run python /workspace/scripts/refresh.py --limit 20 \
  --api-url http://backend:8000
```

First run — all plants processed:

```
INFO done summary={'processed': 8, 'skipped_duplicate': 0, 'total': 8}
```

Run immediately again — Redis idempotency prevents duplicate work:

```
INFO done summary={'processed': 0, 'skipped_duplicate': 8, 'total': 8}
```

## 8. Tests

Run from the repo root with the backend venv active:

```bash
# Backend — 59 tests
cd backend && uv run pytest --cov=app --cov-report=term-missing

# Scripts (CLI + async refresh) — 6 tests
cd .. && backend/.venv/bin/python -m pytest scripts/tests/ -v

# AI service — 3 tests
cd ai_service && ../backend/.venv/bin/python -m pytest tests/ -v

# Frontend — 10 tests
cd ../frontend && .venv/bin/python -m pytest tests/ -v
```

Total: **78 tests**.

### Schemathesis contract check (requires running backend)

```bash
cd backend
uv run pip install schemathesis
uv run schemathesis run http://localhost:8000/openapi.json \
  --checks status_code_conformance --dry-run
```

---

## 9. Changing ports

Edit `.env` before starting:

```ini
BACKEND_PORT=9000
FRONTEND_PORT=9501
AI_PORT=9001
```

Then rebuild:

```bash
sudo docker compose -f compose.yaml up --build
```

---

## 10. Stop and clean up

```bash
# Stop containers, keep the SQLite volume
sudo docker compose -f compose.yaml down

# Stop and delete all data (full reset)
sudo docker compose -f compose.yaml down -v
```
